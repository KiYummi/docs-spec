# 全链路索引策略详解

> **适用范围**：文档工程体系（docs-spec）
> **最后更新**：2026-05-16
> **涉及思想**：索引、全链路追踪

---

## 一、概念

### 1.1 全链路索引

全链路索引是一种需求追踪机制，确保每条需求在全链路中都有唯一标识和可追溯性。

**核心要素**：
- 唯一 ID：每条需求有唯一标识（@req-id: R001）
- 索引文件：全局唯一事实源（index.json）
- 标注机制：所有制品必须标注承接的需求 ID

### 1.2 索引结构

```json
{
  "requirements": {
    "R001": {
      "id": "R001",
      "description": "用户可以通过手机号注册",
      "source": "PRD §1.2",
      "priority": "P0",
      "artifacts": [
        {
          "type": "sub-prd",
          "path": "docs/00_meta/sub-prd.md",
          "section": "## 1. 用户注册"
        },
        {
          "type": "lld",
          "path": "docs/02_backend/specs/GLOBAL_HLD.md",
          "section": "## 3. 用户模块"
        },
        {
          "type": "api",
          "path": "docs/03_api/contracts/user.yaml",
          "endpoint": "/api/v1/users"
        }
      ]
    }
  }
}
```

## 二、实现方式

### 2.1 唯一 ID 分配

**规则**：
- 每条原子需求分配唯一 ID
- ID 格式：`@req-id: R{序号}`
- 序号全局唯一，不重复

**示例**：
- @req-id: R001
- @req-id: R002
- @req-id: R003

### 2.2 索引文件

**位置**：`docs/03_iterations/index.json`

**内容**：
- 需求 ID
- 需求描述
- 来源
- 优先级
- 承接的制品列表

### 2.3 标注机制

**规则**：
- 所有制品必须标注承接的需求 ID
- 标注格式：`@req-id: R{序号}`
- 标注位置：文档头部或相关章节

**示例**：

```markdown
## 需求溯源区

| 原子需求 ID | 语义要素 | 来源 | 推导链 |
|------------|---------|------|--------|
| @req-id: R001 | 用户可以通过手机号注册 | PRD §1.2 | {推理过程} |
```

## 三、索引更新流程

### 3.1 自动更新

**触发时机**：
- PRD 更新时
- Sub-PRD 更新时
- LLD 更新时
- API 合约更新时

**更新内容**：
- 扫描 @req-id 标签
- 更新索引文件
- 检查覆盖率

### 3.2 手动更新

**触发时机**：
- 人确认需求变更时
- 人确认制品变更时

**更新内容**：
- 更新需求描述
- 更新来源
- 更新优先级
- 更新承接的制品列表

---

## 四、索引查询

### 4.1 正向查询

**查询方式**：需求 ID → 制品列表

**示例**：
```
@req-id: R001 → [
  docs/00_meta/sub-prd.md,
  docs/02_backend/specs/GLOBAL_HLD.md,
  docs/03_api/contracts/user.yaml
]
```

### 4.2 反向查询

**查询方式**：制品 → 需求 ID 列表

**示例**：
```
docs/03_api/contracts/user.yaml → [
  @req-id: R001,
  @req-id: R002
]
```

---

## 五、效果

### 5.1 可追溯性

- 每条需求都有唯一标识
- 每个制品都标注了承接的需求
- 可以正向和反向查询

### 5.2 覆盖率检查

- 可以检查需求是否在下游有承接
- 可以检查制品是否标注了需求
- 可以计算覆盖率

### 5.3 偏差检测

- 可以检测需求遗漏
- 可以检测制品缺失
- 可以检测标注错误
