## 二、人/Agent 职责分工

### 2.1 人的职责（人在回路）

| 职责 | 说明 |
|------|------|
| 定义原子需求 | 人确认 PRD 中的原子需求清单 |
| 确认溯源区 | 人确认下游文档的溯源区内容 |
| 签发补丁 | 人审批残差审计的补丁建议 |
| 演化规则 | 人决策规则的新增、修改、删除 |

### 2.2 Agent 的职责

| Agent | 职责 |
|-------|------|
| pm | 提取原子需求，分配唯一 ID |
| docs | 填写溯源区，维护术语一致性 |
| architect | 填写设计文档的溯源区，编写推导链 |
| reviewer | 检查术语一致性 |
| qa | 检查需求覆盖率 |
| residual-auditor | 执行残差审计，输出补丁建议 |
| coordinator | 触发审计，审核结果，签发补丁，记录日志 |

## 四、MOE 团队模型（SP 执行）

```
┌─────────────────────────────────────────────────────────────────┐
│                    MOE 团队模型（SP 执行）                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │              Coordinator（门控网络）                       │   │
│  │    根据 Phase 类型动态选择执行 agent                       │   │
│  └───────────────────────┬─────────────────────────────────┘   │
│                          │                                      │
│       ┌──────────────────┼──────────────────┐                  │
│       ▼                  ▼                  ▼                  │
│  ┌─────────┐      ┌─────────┐      ┌─────────┐               │
│  │   pm    │      │architect│      │  docs   │               │
│  │ (PRD)   │      │ (架构)  │      │ (文档)  │               │
│  └─────────┘      └─────────┘      └─────────┘               │
│                                                                 │
│  ┌─────────┐      ┌─────────┐                                 │
│  │reviewer │      │residual │                                 │
│  │ (审查)  │      │ auditor │                                 │
│  │         │      │ (审计)  │                                 │
│  └─────────┘      └─────────┘                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**门控逻辑**：
- PRD 类型 → 调用 pm
- 架构类型 → 调用 architect
- 技术文档类型 → 调用 docs
- 代码类型 → 调用 backend/frontend
- 审查 → 调用 reviewer
- 审计 → 调用 residual-auditor

**稀疏激活**：每次只激活相关的 agent，不是全部。
