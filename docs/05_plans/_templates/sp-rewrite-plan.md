# SP 执行计划模板 (SP Template)

> **用途**: 定义 SP（Step Plan）的标准结构，用于 AI 辅助开发的任务拆解和执行
> **适用场景**: 单人主导 + AI 辅助编程的敏捷开发模式

---

## 一、SP 结构

```
step_plans/
├── _templates/
│   ├── sp-template.md           # SP 主文档模板（本文件）
│   └── phase-template.md        # Phase 文档模板
├── SP-01_{模块名}/
│   ├── SP-01.md                 # SP 主文档
│   ├── SP-01-Phase-01_{描述}.md  # Phase 文档
│   ├── SP-01-Phase-02_{描述}.md
│   └── ...
├── SP-02_{模块名}/
│   └── ...
```

---

```markdown
|------|--------|

| 依赖项 | 状态 | 说明 |
|--------|------|------|

|-------|------|------|--------|

```

## 四、团队模型

| 角色 | Agent | 职责 |
|------|-------|------|
| 调度 | Coordinator | 任务分配、进度跟踪、质量把关 |
| 执行 | backend/frontend/docs | 编码、文档编写 |
| 审查 | reviewer | 代码审查、文档审查 |
| 审计 | residual-auditor | 残差保真度审计 |

---

## 五、执行流程

```
1. 创建 SP 主文档
   ↓
2. 拆分 Phase 文档
   ↓
3. 逐 Phase 执行：
   a. 执行任务
   b. 更新任务状态
   c. 产出物验收
   ↓
4. SP 完成验收
   ↓
5. 归档
```
