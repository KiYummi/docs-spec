# 测试质量检查清单

测试用例/报告完成后逐条自检，确保覆盖率、缺陷报告质量和红蓝对抗场景覆盖。

## 检查项

### 覆盖率

- [ ] AC 覆盖率 100% — 所有 AC 有自动化测试用例且全绿
- [ ] 契约一致性 — 实际接口返回与 api/contracts/ 定义一致

### 缺陷报告

- [ ] 缺陷信息完整度 — Bug 报告含复现步骤、日志、数据

### 红蓝对抗

- [ ] 极端场景覆盖 — 并发/越权/脏数据有测试用例
- [ ] 脏数据兜底 — 非法输入有处理策略
- [ ] 并发超卖 — 同时操作同一资源有防护
- [ ] 垂直越权 — 普通用户调用管理员接口被拦截
- [ ] 水平越权 — 用户 A 访问用户 B 数据被拦截
- [ ] 边界条件 — 数据量超预期有兜底
- [ ] 事件重复消费 — 重复消息不产生副作用
