# Gate 审计检查清单

Gate 阶段审查时逐条检查，覆盖从架构骨架到发版全链路。

## 检查项

### Phase 2→3 交接审查

- [ ] ER 一致性 — 全局 ER 图与实际设计一致，无遗漏表/字段
- [ ] 枚举/状态机完整性 — 所有状态机已注册，无同名不同义冲突
- [ ] 硬编码扫描 — 流程定义、角色名、待办类型已配置化
- [ ] ACL 映射 — 每个权限点已标注 {平台层} 层来源

### Phase 3 骨架+血肉审查

- [ ] 任务划分完整性 — 未遗漏 PRD 边界
- [ ] AC 覆盖 NFR — AC 涵盖 PRD 的非功能需求
- [ ] API 契约审查 — 幂等性、分页规范、字段粒度、前端可用性、可扩展性、可测试性

### Phase 3 红蓝+架构审查

- [ ] 安全攻击面 — 注入/越权/PII 盘点
- [ ] 多租户隔离 — ORM 自动 tenant_id，禁止手动 WHERE 拼接
- [ ] 性能瓶颈 — 热点查询 <500ms，无 N+1
- [ ] 模块耦合 — 依赖方向正确，无循环依赖
- [ ] 数据一致性 — 乐观锁/幂等性/事务完整性
- [ ] 分层合理性 — Controller/Service/Repository 职责清晰

### Phase 4~6 累积审查

- [ ] L1 委托审查 — API 契约一致性（{reviewer} 执行 L1，本 Agent 负责契约层）
- [ ] L2-a~L2-d — 逐层累积审查要点
- [ ] Phase 5 Gate 9e/9f — 缺陷回归 + UAT 验收
- [ ] Phase 6 — 发版资产 9 项检查

## 红旗

> 停止打勾，先解决：

- 发现硬编码的角色名或状态值 → 先配置化再继续
- ACL 映射缺失 → 权限体系不完整，后续无法验收
- 跨模块状态机不一致 → 可能导致端到端流程断裂
