# PRD 质量检查清单

PRD 完成后逐条自检，确保业务完整性与非功能需求无遗漏。

## 检查项

### 业务完整性

- [ ] 灵魂三问 — 拆解粒度合适？模块解耦干净？数据模型符合标准？
- [ ] 逆向流程 — 退货/冲销/驳回/撤回是否覆盖？
- [ ] 权限定义 — 操作权限 + 数据可见性权限是否完整？
- [ ] 历史脏数据 — 清洗方案？新旧版本兼容？

### 非功能需求

- [ ] 性能指标 — 响应时间上限 + 并发用户数 + 数据量级
- [ ] 多租户隔离方案 — 物理 vs 逻辑选型 + 数据过滤字段
- [ ] 审计日志策略 — 操作范围 + 保留周期 + 脱敏规则

## 红旗

> 停止打勾，先解决：

- PRD 未覆盖逆向流程（退货/冲销） → 先补全再继续
- 多租户隔离方案未明确 → 可能影响全模块设计
- 缺少性能指标量化 → 无法定义测试基线
