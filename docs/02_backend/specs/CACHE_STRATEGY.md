## 1. 缓存分层

| 层 | 技术 | TTL | 场景 |
|---|------|-----|------|
| L1 | 进程内 | {N}s | 热点配置 |
| L2 | Redis | {N}min | 业务数据 |
| L3 | CDN | {N}h | 静态资源 |

## 2. 缓存 Key 规范

```
{tenant_id}:{module}:{entity}:{id}
```

示例：`t123:order:detail:456`

## 3. 缓存策略

| 策略 | 场景 | 说明 |
|------|------|------|
| Cache-Aside | 读多写少 | 先查缓存，miss 后查 DB |
| Write-Through | 强一致 | 写 DB 同时更新缓存 |
| Write-Behind | 高吞吐 | 先写缓存，异步写 DB |
