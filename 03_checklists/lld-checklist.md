# LLD 质量检查清单

> **阶段**: 设计阶段
> **触发时机**: LLD 完成后

---

## 检查项

### 数据模型合规

- [ ] 语义检查 — 实体与字段命名已在术语表中定义
- [ ] ER 一致性 — 新增/修改的表已注册到全局 ER 图
- [ ] 枚举注册 — 新增的枚举值已注册到枚举字典
- [ ] 状态机字典 — 有状态机的实体已注册
- [ ] 防腐字段覆盖 — 所有表包含标准审计字段

### 权限与安全

- [ ] 权限注册 — 每个端点的 permission_code 已注册
- [ ] 安全基线 — 安全要求已链接引用

### NFR 与架构

- [ ] NFR 数值聚合 — 缓存TTL/批量上限/并发阈值已聚合
- [ ] 硬编码扫描 — 流程定义、角色名已配置化
- [ ] 契约完整性 — API 的 JSON 结构与状态码已写入契约文件
- [ ] 架构合规性 — 系统拓扑满足无状态设计

### 需求溯源区

- [ ] 溯源区存在 — LLD 是否包含"需求溯源区"章节？
- [ ] 溯源区完整 — 溯源区是否覆盖所有相关原子需求？
- [ ] 溯源区未篡改 — 溯源区内容是否逐字复制上游原文？
- [ ] @req-id 标注 — 每条需求是否标注了唯一 ID？
- [ ] 推导链完整 — 每个设计决策是否包含推理链？
- [ ] 术语一致 — 字段名是否符合 metadata-dictionary？

## 红旗

> 停止打勾，先解决：

- 新增表未注册到全局 ER → 后续所有 Agent 都会遗漏
- 发现硬编码枚举值 → 改为配置化再继续
- 状态机未注册 → 开发时无法校验转换合法性
- 溯源区缺失 → 无法追溯到原始需求
