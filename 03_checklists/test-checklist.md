# 测试质量检查清单

> **阶段**: 测试阶段
> **触发时机**: 测试完成后

---

## 检查项

### 测试覆盖

- [ ] P0 功能覆盖 — P0 功能是否有 E2E 测试？
- [ ] P1 功能覆盖 — P1 功能是否有集成测试？
- [ ] 边界条件覆盖 — 边界条件是否有测试？
- [ ] 异常场景覆盖 — 异常场景是否有测试？

### 测试质量

- [ ] 测试独立性 — 测试之间是否独立？
- [ ] 测试可重复 — 测试是否可重复执行？
- [ ] 测试数据清理 — 测试后数据是否清理？

### 需求追溯

- [ ] 测试用例追溯 — 每个测试用例是否追溯到需求 ID？
- [ ] 需求覆盖率 — 是否所有需求都有测试覆盖？

## 红旗

> 停止打勾，先解决：

- P0 功能无 E2E 测试 → 补充测试
- 测试之间有依赖 → 重构测试
- 需求无测试覆盖 → 补充测试用例
