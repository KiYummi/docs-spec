# 代码质量检查清单

> **阶段**: 编码阶段
> **触发时机**: 代码提交前

---

## 检查项

### 命名规范

- [ ] 变量命名 — 遵循 glossary.md 术语定义
- [ ] 函数命名 — 使用 CamelCase
- [ ] 常量命名 — 全大写下划线
- [ ] 数据库字段 — snake_case

### 安全底线

- [ ] SQL 注入 — 使用参数化查询或 ORM
- [ ] 用户输入校验 — 类型与边界校验
- [ ] 日志脱敏 — 禁止打印敏感信息
- [ ] 权限校验 — 敏感接口包含越权检查

### 代码质量

- [ ] 错误处理 — 所有错误返回结构化错误码
- [ ] 日志规范 — 关键业务节点记录日志
- [ ] 注释规范 — 复杂逻辑有注释
- [ ] 测试覆盖 — 核心逻辑有单元测试

## 红旗

> 停止打勾，先解决：

- 发现硬编码敏感信息 → 立即移除
- 发现 SQL 拼接 → 改为参数化查询
- 发现吞掉错误 → 添加错误处理
