# API 契约检查清单

> **阶段**: 设计阶段
> **触发时机**: API 契约完成后

---

## 检查项

### 基础规范

- [ ] 路径规范 — 使用 kebab-case，复数形式
- [ ] 方法规范 — GET 查询/POST 创建/PUT 更新/DELETE 删除
- [ ] 响应格式 — 统一 code/message/data 结构
- [ ] 错误码 — 全局错误码已注册

### 需求标注

- [ ] x-req-ids — 每个端点是否标注了需求 ID？
- [ ] 需求覆盖 — 所有原子需求是否在 API 中有承接？

### 安全

- [ ] 认证 — 需要认证的端点是否标注？
- [ ] 权限 — 每个端点是否标注了 permission_code？
- [ ] 限流 — 每个端点是否标注了 rate_limit_level？

### 幂等性

- [ ] 写操作幂等 — 所有写操作是否定义了幂等策略？
- [ ] 幂等键 — 需要幂等键的操作是否标注？

## 红旗

> 停止打勾，先解决：

- 端点缺少 x-req-ids → 无法追溯到需求
- 端点缺少权限标注 → 存在越权风险
- 写操作未定义幂等策略 → 存在重复执行风险
