# PRD 质量检查清单

PRD 完成后逐条自检，确保业务完整性与非功能需求无遗漏。

## 检查项

### 业务完整性

- [ ] 灵魂三问 — 拆解粒度合适？模块解耦干净？数据模型符合标准？
- [ ] 逆向流程 — 退货/冲销/驳回/撤回是否覆盖？
- [ ] 权限定义 — 操作权限 + 数据可见性权限是否完整？
- [ ] 历史脏数据 — 清洗方案？新旧版本兼容？

### 非功能需求

- [ ] 性能指标 — 响应时间上限 + 并发用户数 + 数据量级
- [ ] 多租户隔离方案 — 物理 vs 逻辑选型 + 数据过滤字段
- [ ] 审计日志策略 — 操作范围 + 保留周期 + 脱敏规则

### 原子需求提取（全链路索引）

- [ ] 原子需求清单 — PRD 完成后是否提取了原子需求清单？
- [ ] 唯一 ID 分配 — 每条原子需求是否分配了唯一 ID（@req-id: R001）？
- [ ] 语义要素完整 — 每条需求是否包含：谁、什么、何时、何地、为什么？
- [ ] 来源标注 — 每条需求是否标注了 PRD 的具体章节？
- [ ] 优先级标注 — 每条需求是否标注了优先级（P0/P1/P2）？
- [ ] Grounding — 原子需求是否逐字复制 PRD 原文，未改写？

## 红旗

> 停止打勾，先解决：

- PRD 未覆盖逆向流程（退货/冲销） → 先补全再继续
- 多租户隔离方案未明确 → 可能影响全模块设计
- 缺少性能指标量化 → 无法定义测试基线
