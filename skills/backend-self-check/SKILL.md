# 后端代码自检清单

后端编码完成提交前逐条自检，覆盖安全、正确性、可观测性全维度。

## 检查项

### 安全与数据完整性

- [ ] 输入拦截 — 外部输入经 DTO/Schema 白名单校验，非法输入返回 HTTP 400
- [ ] 注入防护 — SQL 使用参数化查询，无字符串拼接
- [ ] 越权校验 — 每个写操作校验 tenant_id + 资源归属
- [ ] 软删除 — 删除操作使用 is_deleted 标记，非物理删除
- [ ] 审计追踪 — 写操作记录 created_by / updated_at

### 正确性与鲁棒性

- [ ] 乐观锁 — 并发更新使用 version 字段
- [ ] 幂等性 — 重复请求返回相同结果
- [ ] 事务完整性 — 跨表操作在同一事务内
- [ ] 错误处理 — 无 panic / bare error，全部结构化错误码

### 可观测性与性能

- [ ] 结构化日志 — 含 TraceID，敏感字段脱敏
- [ ] 无 N+1 — 关联查询使用 Preload / JOIN

### 契约与工具

- [ ] API 契约一致 — 请求/响应结构与契约完全一致
- [ ] 状态机校验 — 状态转换逻辑匹配定义，含 guard + side_effects
- [ ] 权限码校验 — permission_code 匹配权限注册表
- [ ] 测试全绿 — 测试用例全部通过
- [ ] lint 通过 — 已运行 lint/check

## 红旗

> 停止打勾，先解决：

- 发现硬编码的 tenant_id 或 role 名 → 改为配置/上下文获取
- lint 被拦截 → 先修复再继续
- 测试未全部通过 → 不提交
