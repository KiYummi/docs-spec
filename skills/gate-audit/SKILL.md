# Gate 审计检查清单

阶段审查时逐条检查，覆盖从架构骨架到发版全链路。

## 检查项

### 设计→开发 交接审查

- [ ] ER 一致性 — 全局 ER 图与实际设计一致，无遗漏表/字段
- [ ] 枚举/状态机完整性 — 所有状态机已注册，无同名不同义冲突
- [ ] 硬编码扫描 — 流程定义、角色名、待办类型已配置化
- [ ] ACL 映射 — 每个权限点已标注来源

### 开发阶段 骨架+血肉审查

- [ ] 任务划分完整性 — 未遗漏需求边界
- [ ] AC 覆盖 NFR — AC 涵盖非功能需求
- [ ] API 契约审查 — 幂等性、分页规范、字段粒度、可扩展性

### 安全+架构审查

- [ ] 安全攻击面 — 注入/越权/PII 盘点
- [ ] 多租户隔离 — 自动 tenant_id，禁止手动 WHERE 拼接
- [ ] 性能瓶颈 — 热点查询达标，无 N+1
- [ ] 模块耦合 — 依赖方向正确，无循环依赖
- [ ] 数据一致性 — 乐观锁/幂等性/事务完整性
- [ ] 分层合理性 — Controller/Service/Repository 职责清晰

### 累积审查

- [ ] 契约一致性 — API 契约与代码一致
- [ ] 缺陷回归 — P0/P1 全修
- [ ] 发版资产 — 文档齐备

### 残差保真度审查（全链路索引 + 残差网络）

- [ ] 索引覆盖率 — 所有原子需求是否在下游文档中有承接？
- [ ] 字段合规率 — 字段名是否符合 metadata-dictionary？
- [ ] 术语一致率 — 术语是否符合 glossary？
- [ ] 数据流违例 — 模块依赖是否遵循 global-blueprint？
- [ ] 溯源区完整性 — 下游文档是否包含"需求溯源区"？
- [ ] 人工审核记录 — 人在回路是否确认了关键决策？

## 红旗

> 停止打勾，先解决：

- 发现硬编码的角色名或状态值 → 先配置化再继续
- 权限映射缺失 → 权限体系不完整
- 跨模块状态机不一致 → 可能导致端到端流程断裂
