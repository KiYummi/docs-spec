# 代码审查检查清单

L1/L2 代码审查时逐条检查，按 P0/P1/P2 分层，覆盖安全、数据一致性、正确性、工艺全维度。

## 检查项

### P0 阻断项

**安全**:
- [ ] SQL 注入 — 全部使用参数化查询，无字符串拼接
- [ ] 越权校验 — 每个写操作校验 tenant_id + 资源归属
- [ ] 数据脱敏 — PII 字段日志中脱敏
- [ ] 审计追踪 — 写操作记录审计字段
- [ ] 输入校验 — 外部输入经 DTO/Schema 白名单校验

**数据一致性**:
- [ ] 乐观锁 — 并发更新使用 version 字段
- [ ] 幂等性 — 重复请求返回相同结果
- [ ] 事务完整性 — 跨表操作在同一事务内
- [ ] 状态机校验 — 状态转换匹配定义，无非法转换
- [ ] 并发安全 — 无竞态条件
- [ ] 软删除 — 删除操作使用 is_deleted 标记

### P1 质量项

**正确性**:
- [ ] 契约一致 — 请求/响应结构与契约一致
- [ ] 权限码一致 — permission_code 匹配注册表
- [ ] 逻辑闭环 — 所有分支有明确处理
- [ ] 边界条件 — 空值/零值/最大值有处理

**鲁棒性**:
- [ ] 防御性编程 — 外部输入不信任
- [ ] 错误处理 — 无 panic/bare error
- [ ] 优雅降级 — 外部依赖失败有兜底
- [ ] 重复请求 — 幂等处理

**可观测性**:
- [ ] TraceID — 全链路追踪
- [ ] 结构化日志 — JSON 格式
- [ ] 错误码 — 统一错误码体系

**性能**:
- [ ] 无 N+1 — 关联查询使用 Preload/JOIN
- [ ] 无状态 — 无跨请求内存状态

### P2 工艺项

- [ ] SRP — 单一职责
- [ ] DRY — 无重复逻辑
- [ ] KISS — 无过度复杂实现
- [ ] 可维护性 — 自解释命名，注释只写"为什么"

## 红旗

> 停止打勾，先解决：

- P0 项有任何 ❌ → 阻断，必须修复后重新审查
- 发现 SQL 字符串拼接 → 立即修复，不继续
- 发现硬编码 tenant_id → 改为上下文获取
